Admin Guidelines (Click to expand)
  1. One question tests ONE dimension only
    Pick only one category: EI / SN / TF / JP. Don’t mix multiple dimensions in one scenario.
  2. Keep the scenario question simple and realistic
    Use common student/work situations. Avoid extreme or confusing contexts.
  3. Translate (EN → BM / ZH)
    Write the English (EN) scenario and options first, then click Translate to generate Bahasa Melayu (BM) and 中文 (ZH). After that, review and edit the translations if needed.
  4. Options A–D must be balanced
    Avoid “good vs bad” choices. All options should look reasonable and similar length.
  5. Options must clearly represent both sides of the chosen category
    • EI: E = engage/speak early, I = observe/process first
    • SN: S = facts/steps, N = ideas/patterns
    • TF: T = logic/fairness, F = empathy/harmony
    • JP: J = plan/structure, P = flexible/adapt
  6. Scoring rule (recommended -5 to +5)
    Only the chosen category should have non-zero values. Other columns should be 0.
  7. Quality check before saving
    ✅ One category only? ✅ Balanced options? ✅ Only selected category has scores?
How the system calculates MBTI (from scenario answers)
When a user picks an option, the system adds the option scores into four totals: EI_total, SN_total, TF_total, JP_total. Then it converts totals to letters:
EI_total > 0 → E, else → I
SN_total > 0 → S, else → N
TF_total > 0 → T, else → F
JP_total > 0 → J, else → P
