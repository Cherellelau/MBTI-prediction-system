Admin Guidelines (Click to expand)
  1. Confirm you are editing the correct career
    Check the MBTI Type and Career Key before making changes to avoid updating the wrong record.
  2. Career Key rules (stable identifier)
    Use snake_case only (e.g., ux_researcher, data_analyst). Allowed: a–z, 0–9, _.
    Avoid changing the Career Key after it has been used because it links EN/BM/ZH text records. If you need a new key, create a new entry instead.
  3. URL rules (reference link)
    URL must start with https:// and should be a reliable source (official, O*NET, JobStreet, university/government pages).
    Avoid unstable or login-required links.
  4. English fields are required
    English Career Name and English Career Description must not be empty. BM/ZH are optional but recommended for a better user experience.
  5. Translation (EN → BM/ZH)
    Update EN first, then click Translate to generate BM and ZH for both Career Name and Career Description.
    After generating, review BM/ZH and edit manually if job terms look inaccurate.
  6. Content quality
    Keep descriptions concise: 1–2 sentences. Explain why the career fits the MBTI type (skills, strengths, work style).
    Avoid vague text like “good career” or “suitable job” — include at least one specific reason.
  7. Save & verify
    Click Save Changes, then refresh or re-open the page to confirm the updated EN/BM/ZH content is saved correctly.
